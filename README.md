# Smart Campus

A full-stack web application for managing campus resources and bookings. Built with Spring Boot (backend) and React (frontend) as a PAF group project.

---

## Features

- **Authentication** — Email-based login with auto-registration; role-based access (Admin, Staff, Student)
- **Resource Management** — CRUD for campus facilities (Lecture Halls, Labs, Meeting Rooms, Equipment) with search and filtering
- **Booking System** — Request bookings with automatic conflict detection; approval/rejection workflow for admins
- **Dashboard** — Overview of resources, pending/approved bookings, and recent activity
- **Role-Based Views** — Admins manage all data; students/staff manage their own bookings

## API Endpoints

### Authentication
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/auth/login` | Login with email |
| POST | `/api/auth/login-or-register` | Login or auto-register |
| GET | `/api/auth/users` | Get all users |

### Resources
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/resources` | List all resources |
| GET | `/api/resources/{id}` | Get resource by ID |
| GET | `/api/resources/search` | Search (type, location, capacity, status) |
| POST | `/api/resources` | Create resource (admin) |
| PUT | `/api/resources/{id}` | Update resource (admin) |
| DELETE | `/api/resources/{id}` | Delete resource (admin) |

### Bookings
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/bookings` | List all bookings (admin) |
| GET | `/api/bookings/{id}` | Get booking by ID |
| GET | `/api/bookings/user/{userId}` | Get user's bookings |
| POST | `/api/bookings` | Create a booking |
| PATCH | `/api/bookings/{id}/approve` | Approve booking (admin) |
| PATCH | `/api/bookings/{id}/reject` | Reject booking with reason (admin) |
| PATCH | `/api/bookings/{id}/cancel` | Cancel booking |
| DELETE | `/api/bookings/{id}` | Delete booking |

## Getting Started

### Prerequisites

- Java 25+
- Node.js 18+
- MySQL 8+

### Database Setup

1. Create a MySQL database:
   ```sql
   CREATE DATABASE springcampus;
   ```

2. The schema is auto-generated by Hibernate on first run.

### Backend Setup

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```

2. Update `src/main/resources/application.properties` if needed:
   ```properties
   spring.datasource.url=jdbc:mysql://localhost:3306/springcampus
   spring.datasource.username=root
   spring.datasource.password=root
   ```

3. Run the Spring Boot application:
   ```bash
   ./mvnw spring-boot:run
   ```

   The backend starts on `http://localhost:8080`.

### Frontend Setup

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Start the development server:
   ```bash
   npm run dev
   ```

   The frontend runs on `http://localhost:5173`.

## Booking Workflow

```
Create Booking (PENDING)
        │
        ├── Admin Approves → APPROVED
        │
        ├── Admin Rejects  → REJECTED (with reason)
        │
        └── User Cancels   → CANCELLED
```

Conflict detection prevents overlapping bookings for the same resource.
